Documentação Cerne versão 2.0.0

Considerações Iniciais

O presente documento tem o objetivo de apresentar os conceitos arquiteturais que compõem o Cerne, bem como as formas de utilizá-lo.

A primeira consideração a ser feita é quanto ao controle de versão utilizado, Versionamento Semântico. O Cerne atual se encontra na versão 2.0.0. Estes três números representam o padrão MAJOR.MINOR.PATCH. A mudança em cada um desses números é feita de acordo com a descrição abaixo:

  • versão Maior(MAJOR): quando fizer mudanças incompatíveis na API,
  • versão Menor(MINOR): quando adicionar funcionalidades mantendo compatibilidade, e
  • versão de Correção(PATCH): quando corrigir falhas mantendo compatibilidade.

Mais sobre isso pode ser visto em Versionamento Semântico 2.0.0.

Estrutura Básica de Arquivos

O Cerne é composto pelos seguintes arquivos:

  • assets
  • documentacao
  • elementos

Assets

A primeira pasta assets contém recursos ligados aos componentes visuais do cerne. Portanto tem algumas subpastas como:

  • css, que contém css do cerne e seus elementos
  • lib, que contém as bibliotecas utilizadas

Várias bibliotecas são utilizadas no cerne, como:

  • Bootstrap v3.3.6
  • JQuery v2.1.4
  • JQueryUI v1.12.0
  • Plotly v1.16.2
  • TagsInput v0.8.0
  • Typeahead v0.11.1
  • FreeAspUpload

Dentre elas as mais importantes são as dependências do Cerne, Jquery e o Bootstrap. As demais podem ser utilizadas conforme for necessário.

Elementos

O objetivo dos arquivos dessa pasta é formar e disponibilizar um conjunto de elementos visuais que podem ser utilizados em diferentes contextos. Os elementos são estruturas html que são retornadas através de comandos ASP. Dessa forma, a camada de elementos encapsula o uso de html e javascript e diminui a necessidade de conhecimento nestas duas tecnologias.

Os elementos são genéricos, podendo ser utilizados de diferentes formas. As classes retornam os htmls através do método criar(). Portanto, é responsabilidade das funções que usam essas classes saber o momento, ordem e lugar mais apropriado para inserir o conteúdo html retornado.

OBS: também pode ser usado o método exibir em casos onde o resultado é exibido na tela e não retornado como uma string que representa um html.

É fortemente recomendado que os elementos criados utilizem a classe cAtributosHtml quando for necessário adicionar atributos html como id, class, type etc.

Ao criar os elementos também é importante notar os padrões adotados, como o método criar, explicado anteriormente e o parâmetro valorPadrao. Pois materá a consistência dos elementos.
